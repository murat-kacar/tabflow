# Changelog

This file records user-visible product release notes for TabFlow. It is
kept deliberately separate from the documentation changelog at
`docs/meta/changelog.md`, which tracks the evolution of the
documentation tree.

The split follows the Keep a Changelog convention
(<https://keepachangelog.com/>):

- This file covers product behaviour that a user or operator can
  observe — new features, behaviour changes, bug fixes, security
  notices, deprecations.
- `docs/meta/changelog.md` covers documentation structure changes —
  new sections, rewrites, removals, terminology revisions.

A change that affects both product behaviour and its documentation
lands in both files.

Entries follow Keep a Changelog section headings
(`Added`, `Changed`, `Deprecated`, `Removed`, `Fixed`, `Security`).
Versions follow Semantic Versioning
(<https://semver.org/>).

## Unreleased

Product versioning and the first tagged release will begin after
Refactor 3 lands and the Blazor runtime reaches feature parity with the
previous Next.js runtime. Until then, this section is intentionally
empty; architectural and documentation churn is tracked in
`docs/meta/changelog.md` instead.
